# Project Summary
The project is an advanced Customer Relationship Management (CRM) system designed to enhance business operations through optimized customer interactions, sales tracking, and financial management. It serves as a comprehensive platform that enables businesses to manage customer relationships, analyze sales processes, and generate detailed financial reports. Recent updates have focused on improving the login functionality and addressing authentication issues.

# Project Module Description
The CRM system includes the following functional modules:
1. **Dashboard**: Displays key metrics, reminders for follow-ups, and sales rankings filtered by year/month.
2. **Customer Management**: Manages customer information with advanced search functionalities, including a dropdown for selecting sales representatives and support for custom industry categories.
3. **Sales Tracking**: Maintains sales records with quick access to customer details, limited to assigned customers for sales roles.
4. **Financial Management**: Tracks income and expenses comprehensively, generating financial reports, including monthly deduction rates and export options.
5. **Task Management**: Monitors tasks related to customer interactions.
6. **Employee Management**: Oversees employee records and roles, including secure login capabilities.
7. **Deals Management**: Manages deals with enhanced search capabilities, restricted to deals associated with the logged-in sales employee.
8. **Settings**: Configures system settings and customer code formats.
9. **Operation Logs**: Logs key actions within the system for auditing purposes.
10. **Media Accounts Management**: Manages social media accounts linked to customers.
11. **Permissions Management**: Defines role-based access controls, ensuring sales roles can only view their own customers and deals.
12. **Customer Service Progress Board**: Oversees service progress and tasks with various views.
13. **Login**: Secure login page for employees with improved error handling for authentication.
14. **Callbacks Management**: Manages callback requests from customers.
15. **Authentication**: Handles user authentication processes, including login and session management.
16. **Monthly Deduction Management**: Manages monthly deduction rates, including CRUD operations.
17. **Reports Management**: Generates and exports financial reports in various formats (CSV, Excel, JSON).

# File Description Inventory
- **README.md**: Documentation for backend and frontend setups.
- **alembic/**: Contains files for managing database migrations.
- **core/**: Implements core functionalities like configuration and authentication.
- **data_models/**: JSON files representing various data models used in the application.
- **dependencies/**: Scripts for managing backend dependencies.
- **lambda_handler.py**: Entry point for AWS Lambda functions.
- **main.py**: The main application logic for the backend, with updated CORS handling.
- **middlewares/**: Custom middleware implementations for request handling.
- **models/**: Definitions of data models used in the application.
- **requirements.txt**: Lists Python packages required for the backend.
- **routers/**: Defines API endpoints and their corresponding handlers.
- **schemas/**: Contains Pydantic schemas for data validation.
- **services/**: Contains business logic and service layer implementations.
- **mock_data/**: Contains mock data initialization files for testing and development.
- **frontend/**: Contains all frontend-related files including components, pages, and configurations.

# Technology Stack
- **Backend**: Python, FastAPI, SQLAlchemy, Alembic, asyncpg, bcrypt, python-jose
- **Frontend**: React, TypeScript, Vite, Tailwind CSS, Recharts

# Usage
1. **Install Dependencies**:
   - For backend: Run `pip install -r requirements.txt`.
   - For frontend: Run `npm install` or `pnpm install`.
   
2. **Build**:
   - For backend: Execute `python main.py`.
   - For frontend: Run `npm run build` or `pnpm build`.

3. **Run**:
   - For backend: Start the server using `uvicorn main:app --reload`.
   - For frontend: Start the development server using `npm run dev` or `pnpm dev`.
